Program Description

This program is an example of steps to allow embedded database connectivity to a C++ program. The following has been checked:

  • Downloading, building and linking Poco library
  • Downloading and linking SQLite library
  • Using different aspects of Poco::Data wrapper according to Poco Data User Guide. Particularly, the following interesting features has been tested:
    • connection, session, binding custom objects
    • using Poco data types (incl. timestamps)
    • some peculiarities of SQLite3: foreign key support

Building Tips

  • POCO library is needed: Download complete edition
  • SQLite library is needed: Download 64-bit DLL (x64) and command-line tools
  • Set environmental variables: CPP_POCO_LIB_ROOT, CPP_SQLITE_LIB_ROOT
  • Use your modified assets/setenv.bat (run as admin) to accomplish that
  • Build POCO on x64 (on win32 it builds without problems):
    1. in Developer Command Prompt (comspec% /k ""C:.bat"); cd C:\; run vc/vcvarsall.bat amd64
    2. cd poco_dir
    3. in component: delete not needed components
    4. in build_vs140.cmd: buildwin 140 build shared both x64 samples
    5. run build_vs140.cmd
  • Set Project Properties
    1. Configuration Properties > Debugging > Environment
      • PATH=PATH%;H:\\;H:\\;
      • these are paths to dll libs PocoFoundation64, PocoData64, PocoDataSQLite64, sqlite3
    2. Configuration Properties > General > Output and Intermediate Directories
      • build\\\bin\
      • build\\\obj\
    3. C/C++ > General > Additional Include Directories
      • H:\\include\;H:\\include\;H:\\SQLite\include\;
    4. Linker > General > Additional Library Directories
      • H:\\;
